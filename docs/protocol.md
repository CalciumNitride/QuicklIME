# TSF層 - エンジン間プロトコル

作成日: 2026-07-13 (フェーズ3)

## トランスポート

- named pipe: `\\.\pipe\quicklime-engine`
- エンジン (quicklime-engine.exe) がサーバ、各アプリにロードされた TSF DLL がクライアント
- 接続は複数同時に成立する (アプリごとに1本)
- TSF 側はエンジンに接続できない場合、フォールバック (ひらがな1候補のみ) で動作を継続する

## メッセージ形式

- 1リクエスト = 1行 (LF 終端)、UTF-8、フィールドはタブ区切り
- 1リクエストに対して必ず1行のレスポンスを返す

### CONVERT: かなを全文一括の変換候補リストへ

```
要求:   CONVERT\t<かな>\n
成功:   OK\t<候補1>\t<候補2>\t...\n
失敗:   ERR\t<メッセージ>\n
```

候補の並び: [Viterbi による文変換] → [短縮よみ (ユーザ辞書、読み完全一致)] →
[読み全体の辞書完全一致 (コスト順)] → カタカナ → ひらがな。

### CONVSEG: かなを文節列へ (フェーズ4-3)

```
要求:   CONVSEG\t<かな>\n
成功:   OK\t<文節1>\t<文節2>\t...\n
失敗:   ERR\t<メッセージ>\n

文節 = <読み>\x1f<候補1>\x1f<候補2>...  (\x1f = ASCII Unit Separator)
```

- 文節は Viterbi 最小コスト経路を品詞で区切ったもの (付属語は前の自立語に付く)
- 各文節の候補は [経路上の表記] → [短縮よみ (ユーザ辞書、文節読み完全一致)] →
  [文節読みの辞書候補 (コスト順)] → カタカナ → ひらがな
- 学習済みの表記がある文節では、その表記が候補の先頭に移動する (短縮よみより優先)

### CONVSYM: 特殊変換の候補取得 (F4 用)

```
要求:   CONVSYM\t<かな>\n
成功:   OK\t<候補1>\t<候補2>\t...\n   (該当なしなら OK のみ)
失敗:   ERR\t<メッセージ>\n
```

- 読みに対応する記号辞書 (symbol.tsv) のエントリ (記載順) と、
  日付・時刻の動的候補をこの順で返す
- 日付・時刻の動的候補: 読みが「きょう」「あした (あす)」「あさって」「きのう」
  「おととい」「ことし」「らいねん」「きょねん」「いま」のとき、エンジンの
  現在日時から作った表記 (2026/07/15、7月15日(水)、令和8年7月15日、14:30 など)
- 通常語・カタカナ・ひらがな候補は含めない
- 該当する候補が無い読みはエラーにせず、候補ゼロの OK を返す

### CONVUSER: ユーザ辞書変換の候補取得 (F5 用)

```
要求:   CONVUSER\t<かな>\n
成功:   OK\t<候補1>\t<候補2>\t...\n   (該当なしなら OK のみ)
失敗:   ERR\t<メッセージ>\n
```

- 読みに完全一致するユーザ登録語 (短縮よみ → 名詞系、それぞれ記載順) の表記のみを返す
- 通常語・記号・カタカナ・ひらがな候補は含めない
- 該当する候補が無い読みはエラーにせず、候補ゼロの OK を返す

### PREDICT: 予測入力の候補取得 (フェーズ4-11)

```
要求:   PREDICT\t<かな>\n
成功:   OK\t<読み>\x1f<表記>\t<読み>\x1f<表記>...\n   (候補なしは OK のみ)
失敗:   ERR\t<メッセージ>\n   (かなの欠落時のみ)
```

- 読みが <かな> で始まる候補を、ユーザ辞書 (短縮よみ → 名詞系、記載順) →
  確定履歴 (新しい順) → システム辞書 (コスト順) の順で最大8件 (設定
  max_predictions で 1-8 に変更可) 返す。
  表記の重複は先勝ちで除き、表記が入力かなそのままの候補は除外する
- 完全一致で最大数に満たない場合、読み3文字以上ならタイプミス補正の曖昧一致
  (1かな置換・隣接かな入れ替え・1かな脱落/余分。先頭かなは対象外) の候補を
  コスト順で末尾に補充する (フェーズ4-15。設定 typo_correction で無効化可)
- 各候補は「読み\x1f表記」のペア。読みは入力の接頭辞ではなく候補の完全な読みで、
  TSF 層が採用時の LEARN にそのまま使う
- かなが2文字 (設定 min_suggest_chars で 1-5 に変更可) 未満・該当なし・
  サジェスト無効 (設定 suggest=0) はエラーにせず、候補ゼロの OK を返す

### LEARN: 確定結果の学習 (フェーズ4-4)

```
要求:   LEARN\t<読み>\x1f<表記>\t<読み>\x1f<表記>...\n
成功:   OK\n
失敗:   ERR\t<メッセージ>\n
```

- TSF 層が変換確定時に、文節ごとの (読み, 選択した表記) を送る
- エンジンは「読み → 最後に確定した表記」として記録し、
  %APPDATA%\QuicklIME\learning.tsv に追記保存する (QUICKLIME_LEARN_FILE で上書き可)
- 例外: 日付・時刻の動的候補と一致する表記は記録しない
  (時間が経つと古くなり、翌日以降も昨日の日付が先頭に来てしまうため)
- 学習が無効 (設定 learning=0) のときは記録せず OK を返す
  (既存の学習データは引き続き候補順の調整に使う)

### ADDWORD: ユーザ辞書への単語登録 (単語登録ツール用)

```
要求:   ADDWORD\t<読み>\t<表記>\t<品詞>\n
成功:   OK\n
失敗:   ERR\t<メッセージ>\n   (引数不足・未対応の品詞・登録済み)
```

- ユーザ辞書ファイル (userdict.tsv) へ「読み\t表記\t品詞」を追記し、
  メモリへも即時反映する (エンジンの再起動なしで変換・予測に効く)
- 品詞は「短縮よみ」と名詞系 (名詞/固有名詞/人名/姓/名/地名/組織) に対応する。
  名詞系は id.def から文脈IDを解決し、Viterbi のラティスに通常語として載る
- 同じ (読み, 表記) の登録済みエントリがあればエラーを返す

### RELOADUSER: ユーザ辞書の再読込

```
要求:   RELOADUSER\n
成功:   OK\n
```

- userdict.tsv を読み直す (手動編集をエンジンの再起動なしで反映する用)

### RELOADCONFIG: 設定ファイルの再読込 (設定ツール用)

```
要求:   RELOADCONFIG\n
成功:   OK\n
```

- config.tsv (%APPDATA%\QuicklIME\config.tsv、QUICKLIME_CONFIG_FILE で上書き可)
  を読み直し、エンジン側の設定 (learning・suggest・typo_correction・
  max_predictions・min_suggest_chars) を即時反映する
- ファイルが無い・不正な行がある場合も既定値で動作を続け、OK を返す
- TSF 層向けのキー (space, punctuation, candidate_font, key.* など) は
  エンジンでは無視される (TSF 層が同じファイルを自前で読む)

## 設計メモ

- タブと改行はかな文字列に現れないため、エスケープは不要
- TSF 側の読み取りは同期・ブロッキング (フェーズ5でタイムアウトを検討)
- プロトコルの拡張はコマンド (先頭フィールド) の追加で行う
