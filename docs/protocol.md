# TSF層 - エンジン間プロトコル

作成日: 2026-07-13 (フェーズ3)

## トランスポート

- named pipe: `\\.\pipe\quicklime-engine`
- エンジン (quicklime-engine.exe) がサーバ、各アプリにロードされた TSF DLL がクライアント
- 接続は複数同時に成立する (アプリごとに1本)
- TSF 側はエンジンに接続できない場合、フォールバック (ひらがな1候補のみ) で動作を継続する

## メッセージ形式

- 1リクエスト = 1行 (LF 終端)、UTF-8、フィールドはタブ区切り
- 1リクエストに対して必ず1行のレスポンスを返す

### CONVERT: かなを全文一括の変換候補リストへ

```
要求:   CONVERT\t<かな>\n
成功:   OK\t<候補1>\t<候補2>\t...\n
失敗:   ERR\t<メッセージ>\n
```

候補の並び: [Viterbi による文変換] → [読み全体の辞書完全一致 (コスト順)] → カタカナ → ひらがな。

### CONVSEG: かなを文節列へ (フェーズ4-3)

```
要求:   CONVSEG\t<かな>\n
成功:   OK\t<文節1>\t<文節2>\t...\n
失敗:   ERR\t<メッセージ>\n

文節 = <読み>\x1f<候補1>\x1f<候補2>...  (\x1f = ASCII Unit Separator)
```

- 文節は Viterbi 最小コスト経路を品詞で区切ったもの (付属語は前の自立語に付く)
- 各文節の候補は [経路上の表記] → [文節読みの辞書候補 (コスト順)] → カタカナ → ひらがな
- 学習済みの表記がある文節では、その表記が候補の先頭に移動する

### CONVSYM: 特殊変換の候補取得 (F4 用)

```
要求:   CONVSYM\t<かな>\n
成功:   OK\t<候補1>\t<候補2>\t...\n   (該当なしなら OK のみ)
失敗:   ERR\t<メッセージ>\n
```

- 読みに対応する記号辞書 (symbol.tsv) のエントリ (記載順) と、
  日付・時刻の動的候補をこの順で返す
- 日付・時刻の動的候補: 読みが「きょう」「あした (あす)」「あさって」「きのう」
  「おととい」「ことし」「らいねん」「きょねん」「いま」のとき、エンジンの
  現在日時から作った表記 (2026/07/15、7月15日(水)、令和8年7月15日、14:30 など)
- 通常語・カタカナ・ひらがな候補は含めない
- 該当する候補が無い読みはエラーにせず、候補ゼロの OK を返す

### PREDICT: 予測入力の候補取得 (フェーズ4-11)

```
要求:   PREDICT\t<かな>\n
成功:   OK\t<読み>\x1f<表記>\t<読み>\x1f<表記>...\n   (候補なしは OK のみ)
失敗:   ERR\t<メッセージ>\n   (かなの欠落時のみ)
```

- 読みが <かな> で始まる候補を、確定履歴 (新しい順) → システム辞書 (コスト順) の順で
  最大8件返す。表記の重複は履歴優先で除き、表記が入力かなそのままの候補は除外する
- 各候補は「読み\x1f表記」のペア。読みは入力の接頭辞ではなく候補の完全な読みで、
  TSF 層が採用時の LEARN にそのまま使う
- かなが2文字未満・該当なしはエラーにせず、候補ゼロの OK を返す

### LEARN: 確定結果の学習 (フェーズ4-4)

```
要求:   LEARN\t<読み>\x1f<表記>\t<読み>\x1f<表記>...\n
成功:   OK\n
失敗:   ERR\t<メッセージ>\n
```

- TSF 層が変換確定時に、文節ごとの (読み, 選択した表記) を送る
- エンジンは「読み → 最後に確定した表記」として記録し、
  %APPDATA%\QuicklIME\learning.tsv に追記保存する (QUICKLIME_LEARN_FILE で上書き可)
- 例外: 日付・時刻の動的候補と一致する表記は記録しない
  (時間が経つと古くなり、翌日以降も昨日の日付が先頭に来てしまうため)

## 設計メモ

- タブと改行はかな文字列に現れないため、エスケープは不要
- TSF 側の読み取りは同期・ブロッキング (フェーズ5でタイムアウトを検討)
- プロトコルの拡張はコマンド (先頭フィールド) の追加で行う
